2. Technical Understanding
In Linux Git environments, branches allow multiple "parallel universes" of a project to exist simultaneously.
    Remote vs. Local: When we clone, we often only "see" the master branch locally. The other branches (remotes/origin/...) are known to Git but not active in our folder until we explicitly checkout.
    Checkout Logic: Running git checkout dev tells the Linux filesystem to swap out the current version of files with the versions stored in the dev branch's database.

3. Key Takeaway
Branching is a common hiding spot. In security auditing, "Production" code (Master) is often cleaned, but "Development" or "Staging" branches (Dev/Sploits) frequently contain sensitive leftovers like hardcoded credentials, debug logs, or internal notes.
Linux Command	Purpose
git branch -a	Lists all branches (local and remote) to find hidden development paths.
git checkout [name]	Physically changes the files in our directory to a different branch's state.
git log -p	Shows the "patch" or specific code changes for every commit in history.

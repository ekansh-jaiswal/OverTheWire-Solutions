1. The Summary of Actions
    Enumeration: we ran git tag to list all tags in the current repository.
    Discovery: we identified a tag named secret that was not visible in the standard file list (ls) or the commit history (git log).
    Extraction: we used git show secret to display the data associated with that tag, which contained the password.
2. Technical Understanding
In Linux Git management, a Tag is a permanent "bookmark" or reference to a specific point in history.
    Why it was hidden: A tag is a reference to a Git object (like a commit). It doesn't necessarily create a new file in our working directory. If we only look at the files we can cat, we will miss data stored in these references.
    The git show command: In a Linux terminal, git show is a versatile tool. When used on a tag, it doesn't just show the code; it shows the tagging metadata, including the tagger's name, date, and any message (or in this case, the password) attached to it.

3. Key Takeaway
Git is a database, not just a folder. In a security or "wargame" context on Linux, always check the References (refs/tags or refs/remotes) because they can hold data that was never meant to be "checked out" into a physical file.
Command	Linux/Git Purpose
git tag	Lists all "bookmarks" in the repository metadata.
git show [tag]	Inspects the contents and metadata of a specific reference.
ls -a	Shows hidden files, but cannot show Git tags.

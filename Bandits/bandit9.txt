### Solution : 
The password is hidden inside a "data.txt" file which contains binary contents as well,
and the whole data looks to be obfuscated in side. 

Hint provided : Data is preceded by several "==" symbols. 

We use the following command : "strings your_file.bin | grep -oP '={2,}\s*\K\w+'"

Here : 

strings: Scans the binary file and outputs only the human-readable text.

grep -oP:

    -o: Prints only the matching part (not the whole line).

    -P: Uses Perl-style regex (allows the \K trick).

    '={2,}\s*\K\w+': Matches 2 or more =, ignores them (\K), then grabs the word (\w+).

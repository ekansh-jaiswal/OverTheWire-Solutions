### Solution :

We are directly logged in into a custom shell which would convert all our lowercase [:lower] into [:UPPER]. 
But we noticed that this doesn't applies to Special Symbols and Numbers and they are excluded. And here we use the concept of Linux Process Trees Now. 
The thought process comes from understanding the Linux Process Tree.
When we log into an SSH session, the server starts a process. Usually, that process is /bin/bash. 
However, in our case, the server started a wrapper script (the "Uppercase Shell").
In Linux shell scripting, $0 is the variable that holds the name of the "calling" shell or the script itself. 
If we are inside a script named jail.sh, then $0 equals jail.sh. But if that script was executed by a shell, or if the shell is running interactively, 
$0 often points to the binary that started the environment.

What exactly does $0 do?
Technically, $0 is a Positional Parameter.
    In a Script: It is the path used to invoke the script.
    In an Interactive Shell: It is the name of the shell itself (e.g., -bash or sh).
Why it is a "Magic Bullet" for our Uppercase Trap:
    Case Immunity: The variable $0 contains symbols (a dollar sign and a zero). our "Uppercase Shell" likely only changes letters (a-z to A-Z). Symbols and numbers are ignored.
    Execution: When we type $0 at a prompt and hit Enter, the shell "expands" the variable first.
        we type: $0
        The shell sees: /bin/sh (or whatever the shell name is)
        The shell then tries to execute that result.
    The Result: we are essentially telling the restricted shell: "Please run a fresh version of ourself." 
    Because the fresh version is a standard Linux shell, it starts without the "uppercase-only" restriction active.

Other "Special" Linux Parameters

Since we are looking at $0, there are a few others that every Linux user should know for troubleshooting:
Parameter	Meaning
$0	        The name of the shell or the current script.
$1,         $2...	The first, second, etc., arguments passed to a command.
$#	        The number of arguments passed.
$?	        The Exit Status of the last command (0 = success, non-zero = error).
$$	        The Process ID (PID) of the current shell.
$USER	    The name of the current logged-in user.

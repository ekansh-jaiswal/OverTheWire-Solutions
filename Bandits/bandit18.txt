### Solution : 

Someone has modified the ~/.bashrc file as follows : 
'''
tail -n 10 ~/.bashrc
# sources /etc/bash.bashrc).
if ! shopt -oq posix; then
  if [ -f /usr/share/bash-completion/bash_completion ]; then
    . /usr/share/bash-completion/bash_completion
  elif [ -f /etc/bash_completion ]; then
    . /etc/bash_completion
  fi
fi
echo 'Byebye !'
exit 0
'''

Due to this the moment we ssh into, we get logged out.
To solve this, we simply don't allow the bashrc profile to load at the first place 
by using the following command as shown : 
	"ssh bandit18@bandit.labs.overthewire.org -p 2220 "bash --norc""

Here by saying "bash --norc" we tell bash not to read the ~/.bashrc file.

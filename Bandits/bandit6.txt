### Solution : 
Here the password for bandit7 was hidden somewhere else than said. 
But the file had the following specifications : 
	1. Owned by bandit7
	2. Owned by group bandit6
	3. Has size of 33 bytes

This info can be used to find the file using the "find" function as shown : 
bandit6@bandit:/home/bandit7$ find / -user bandit7 -group bandit6 -size 33c 2>/dev/null
/var/lib/dpkg/info/bandit7.password
bandit6@bandit:/home/bandit7$ cat /var/lib/dpkg/info/bandit7.password
morbNTDkSW6jIlUc0ymOdMaLnOlFVAaj
bandit6@bandit:/home/bandit7$

Here : 

- find /: Tells the system to search everywhere, starting from the root directory (/) and looking through every single subfolder on the server.
- user bandit7: Filters the results to only show files owned by the user bandit7.
- group bandit6: Further filters to only show files assigned to the group bandit6 (which is you).
- size 33c: Specifically looks for a file that is exactly 33 bytes ("c" stands for characters/bytes).
- 2>/dev/null: This is the "cleaner."
    i. Since we are searching the whole server, you’ll encounter thousands of folders you don't have permission to enter.
    ii. Normally, the terminal would scream Permission denied for every single one, burying your answer in "noise."
    iii. This part sends those error messages (stream 2) to a "black hole" (/dev/null), so you only see the actual file path you're looking for.
